feat(iam): 实现完整的 OAuth2 授权服务器

实现 OAuth2 Provider 的完整功能，支持授权码流程和 PKCE。

Proto 定义：
- proto/iam/oauth.proto - OAuth2 gRPC 服务定义
- 10 个 RPC 方法（Client 管理 + 授权流程）

基础设施层（4 个 Repository）：
- PostgresOAuthClientRepository - Client 管理
- PostgresAuthorizationCodeRepository - 授权码管理
- PostgresAccessTokenRepository - Access Token 管理
- PostgresRefreshTokenRepository - Refresh Token 管理
- 完整的租户隔离支持

领域服务：
- OAuthService - 核心业务逻辑
- 授权码生成和验证
- Token 交换（授权码换 Token）
- Token 刷新
- Token 撤销和验证
- PKCE 支持（S256/plain）

应用层：
- CreateClientCommand/Handler - 创建 OAuth Client
- AuthorizeCommand/Handler - 授权请求
- TokenCommand/Handler - Token 请求
- 完整的 CQRS 模式

API 层：
- OAuthServiceImpl - gRPC 服务实现
- OAuth Client 管理（CRUD）
- 授权端点（/authorize）
- Token 端点（/token）
- Token 撤销（/revoke）
- Token 验证（/introspect）

核心功能：
- 授权码流程（Authorization Code Flow）
- PKCE 支持（防止授权码拦截攻击）
- Token 刷新机制
- Token 撤销
- 完整的租户隔离
- 公开客户端和机密客户端支持

安全特性：
- 32 字节随机 Token 生成
- PKCE code_challenge 验证（S256/plain）
- 授权码一次性使用（10 分钟过期）
- Access Token 过期管理（1 小时）
- Refresh Token 过期管理（7 天）
- 完整的租户隔离
- Client Secret 哈希存储

文件统计：
- 新增文件：20+ 个
- Repository 实现：4 个
- 应用层 Handler：3 个
- gRPC 服务：1 个完整实现
